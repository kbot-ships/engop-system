# Contributing

This framework is meant to evolve with the teams that use it. Contributions welcome.

## What to contribute

- **Case studies** showing contract-first applied to a real feature your team shipped
- **Template improvements** based on what worked (or didn't) when you used them
- **Automation examples** -- webhook listeners, CI integrations, bot implementations
- **Decision records** from teams adopting the framework

## How to contribute

1. Fork the repo
2. Create a branch (`feature/your-change`)
3. Make your changes
4. Open a PR with a short description of what changed and why

## Guidelines

- Keep documents concise. If it's longer than 2 pages, it's probably two documents.
- Include real examples, not hypotheticals. Anonymize if needed, but keep it concrete.
- Templates should be copy-paste ready. No setup required.
- Test SVGs with `xmllint --noout` before pushing (GitHub's SVG sanitizer is strict).

## Questions?

Open an issue. We'd rather answer a question than fix a misunderstanding later.
